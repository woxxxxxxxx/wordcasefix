PascalCase Converter: Classes, Components, and Type Names

PascalCase (UpperCamelCase) capitalizes every word including the first: UserProfile, ApiResponse, ButtonProps. It's the standard for class names in virtually every object-oriented language — Java, C#, TypeScript, Python, Swift — and is required by React for component names (a component named userCard will not be recognized; it must be UserCard).

This converter takes any input — plain text, snake_case, kebab-case, camelCase, or mixed — and outputs clean PascalCase. Type "user profile settings" and get UserProfileSettings. Type handle-form-submit and get HandleFormSubmit. It's the fastest way to generate consistent class names, interface names, type aliases, and enum values when building in a statically typed language.

PascalCase is also used in Go for exported identifiers — any function or variable starting with a capital letter is automatically public (exported). In C#, PascalCase applies to methods, properties, and events as well as class names. Knowing when to use PascalCase vs camelCase is one of the first conventions new developers learn, and it varies subtly by language. This tool handles the conversion; our guide covers the conventions.
