How to Use This Tool

  1. Select mode: Encode turns plain text into percent-encoded URL-safe text; Decode reverses it.
  2. Paste your text or URL into the input field — the conversion runs as you type.
  3. Click "Copy Result" to put the encoded or decoded output on your clipboard. Use the swap button to reverse direction.

Common Use Cases

  • Building query strings: Safely embed user input, search terms, or special characters (spaces, &, =, /) into URL parameters.
  • Debugging API requests: Decode the URL parameters from a webhook payload, OAuth callback, or browser address bar to see the original values.
  • Sharing links with special characters: Encode URLs containing non-ASCII characters (e.g. Chinese, emoji) before pasting them into chat, email, or QR codes.

Frequently Asked Questions

What's the difference between encodeURI and encodeURIComponent?
This tool uses encodeURIComponent, which escapes every reserved URL character (including /, ?, &, =). Use it when encoding a value inside a query string. encodeURI leaves those reserved characters alone — use it only when encoding a whole URL.
Why is a space sometimes %20 and sometimes +?
%20 is the URL-path encoding; + is used in application/x-www-form-urlencoded query strings. This tool decodes both, but always encodes to %20 for maximum compatibility.
Will it handle non-ASCII characters like 中文 or emoji?
Yes. Characters are encoded using UTF-8 byte sequences, the standard for modern URLs. So 中 becomes %E4%B8%AD and round-trips back cleanly on decode.
